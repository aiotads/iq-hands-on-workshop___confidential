開始前請注意：本實作流程僅支援 Windows 11，請務必使用 Win11 系統進行操作。 Please note: This tutorial only supports Windows 11. Please use a Windows 11 system for the following steps.
01

Qualcomm Software Center (QSC)

下載 PCAT / QUDDownload PCAT / QUD
  1. 下載並安裝 QSC：Download and install QSC: https://softwarecenter.qualcomm.com/
  2. 安裝後以 Qualcomm 帳號登入。After installing, sign in with your Qualcomm account.
02

PCAT Tool

燒錄 image 工具Image flashing tool
  1. 打開 Qualcomm Software Center。Open Qualcomm Software Center.
  2. 在 Tools 搜尋 Product Configuration Assistant Tool，下載並安裝。Under Tools, search Product Configuration Assistant Tool, then download and install.
03

QUD (Qualcomm USB Driver)

Qualcomm USB 驅動Qualcomm USB driver
  1. 打開 Qualcomm Software Center。Open Qualcomm Software Center.
  2. 在 Tools 搜尋 Qualcomm USB Driver，下載並安裝。Under Tools, search Qualcomm USB Driver, then download and install.
小提醒:一次只能安裝一個工具，若畫面顯示 No logs found 訊息，代表該工具尚未安裝。 Note: Please install only one tool at a time. If the No logs found message appears, it indicates that the tool has not been successfully installed.
04

WSL + Ubuntu Setup

安裝環境為當日實作(Hands-on的基礎)，請務必於課前完成配置。 A pre-configured environment is essential for the hands-on session; please ensure the installation is successful beforehand.
  1. 以管理員權限開啟 Windows PowerShell，執行以下指令。 Open Windows PowerShell with Administrator rightsand execute the following command.
    bash 
    wsl.exe --install -d Ubuntu-22.04 --no-launch
  2. 執行以下指令，檢查是否安裝成功。 Run the following command to check if the installation was successful.
    bash 
    wsl.exe --list --verbose
PowerShell顯示Ubuntu-22.04就可以了!! As long as PowerShell displays 'Ubuntu-22.04', you're good to go!
若安裝過程中遇到權限限制，請聯繫貴公司的 IT 或 MIS 部門協助排除。 After installing, QSC showing you have no devices connected is normal (no board attached yet).
wsl.exe --list --verbose 沒看到 Ubuntu-22.04，或安裝過程中出現錯誤？點開查看解法 Don't see Ubuntu-22.04 in wsl.exe --list --verbose, or hit an error during install? Click to see the fix

請依序執行以下指令：Run the following commands in order:

  1. 查看可安裝的發行版清單。 List the available distros online.
    bash 
    wsl --list --online
  2. 更新 WSL 核心元件。 Update the WSL core components.
    bash 
    wsl --update
  3. 改用網頁下載方式重新安裝 Ubuntu-22.04。 Reinstall Ubuntu-22.04 using web download instead.
    bash 
    wsl --install -d Ubuntu-22.04 --web-download --no-launch
