BYOM 模型從 Float32 到 INT8 的裝置端部署 Qualcomm AI Hub ✕ YOLO26：
BYOM from float32 to int8 On-Device Deployment

在 Windows 11 + WSL2 主機與 EXMP-Q911（Qualcomm QCS9075）裝置上，走一遍完整的 YOLO26 模型部署流程。 Walk through the full YOLO26 model deployment flow on a Windows 11 + WSL2 host with an EXMP-Q911 (Qualcomm QCS9075) target device. — just run them in order. No manual command typing is required.

Windows 主機設定（僅需一次）Windows Host Setup (one-time)

[WIN-PS]

以系統管理員開啟 Windows PowerShell（開始 → 搜尋 PowerShell → 右鍵 → 以系統管理員身分執行），切換到套件的 scripts 資料夾： Open Windows PowerShell as Administrator (Start → search for PowerShell → right-click → Run as administrator), then change into the package's scripts folder:

powershell 
cd (Get-ChildItem "$env:USERPROFILE" -Directory -Recurse -ErrorAction SilentlyContinue | Where-Object { $_.FullName -like "*hands_on_workshop_package*\scripts" } | Select-Object -First 1 -ExpandProperty FullName)

執行 Windows 端的設定腳本（約需 5–10 分鐘）： Run the Windows-side setup script (takes about 5–10 minutes):

powershell 
powershell -ExecutionPolicy Bypass -File .\setup-windows-wsl.ps1

此腳本會安裝並設定 WSL2 Ubuntu-22.04、啟用 systemd，並透過 usbipd 把 EXMP-Q911 的 USB 裝置掛進 WSL。過程需要互動： The script installs and configures WSL2 Ubuntu-22.04, enables systemd, and attaches the EXMP-Q911 USB device into WSL via usbipd. The setup is interactive:

  • Windows 詢問是否繼續時，回答 Y 或 Yes。When Windows asks whether to continue, answer Y or Yes.
  • 若是第一次初始化 Ubuntu-22.04，出現提示時請建立 WSL 帳號（設定使用者名稱與密碼）。If Ubuntu-22.04 is being initialized for the first time, create the WSL account when prompted (set a username and password).
完成後會自動開啟一個 WSL Ubuntu 終端機。之後所有步驟都在 WSL 內執行。 When it finishes, a WSL Ubuntu terminal opens automatically. All remaining steps run inside WSL.
