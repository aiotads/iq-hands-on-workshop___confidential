adb 使用方式Using adb

adb（Android Debug Bridge）用來從 Windows 連到平台下指令、進入 shell。前提：已安裝 platform-tools（見「預先安裝說明」）。 adb (Android Debug Bridge) lets you run commands and enter a shell on the platform from Windows. Prerequisite: platform-tools is installed (see "Pre-install guide").

連線與進入 shellConnect and enter the shell
  1. 用 type-c 線接到平台的 Flash / ADB port，並確認平台已開機完成。Connect a USB-C cable to the platform's Flash / ADB port, and confirm the platform has finished booting.
  2. 開啟 PowerShell，切換到 platform-tools 資料夾。Open PowerShell and change into the platform-tools folder.
  3. 執行 .\adb.exe devices，確認列出裝置序號且狀態為 device。Run .\adb.exe devices and confirm the device serial is listed with status device.
  1. 進入裝置 shell：執行 .\adb.exe shell，出現 sh-5.2# 提示字元即成功。Enter the device shell: run .\adb.exe shell; the sh-5.2# prompt means success.
常見問題（debug）Common issues (debug)
  • 「adb devices」看不到裝置（清單空白）："adb devices" shows no device (empty list):
    • 確認 type-c 接在 Flash / ADB port（非其他 USB 孔）。Confirm USB-C is on the Flash / ADB port (not another USB port).
    • 確認平台已完全開機Confirm the platform has fully booted.
    • 換一條 type-c 線或換 USB port 再試。Try another USB-C cable or USB port.
  • 裝置顯示 unauthorized 或 offline：重新插拔Device shows unauthorized or offline: unplug and replug.

MobaXterm (UART) 使用方式Using MobaXterm (UART)

UART 是最底層的 debug 管道：開機卡住、無畫面、adb 連不上時，都可用它觀察平台的開機 log。 UART is the lowest-level debug channel: when boot hangs, there's no display, or adb won't connect, use it to watch the platform's boot log.

前置：安裝 MobaXterm 與 UART 驅動Setup: install MobaXterm and the UART driver
  1. 到官網下載 MobaXterm Home Edition（Installer edition）：Download MobaXterm Home Edition (Installer edition) from the official site: https://mobaxterm.mobatek.net/download-home-edition.html
找出 COM portFind the COM port
  1. 用 USB-to-UART 線連接平台的 debug 接腳與電腦。Connect the platform's debug pins to the computer with a USB-to-UART cable.
  2. 開啟「裝置管理員」→ 展開「連接埠 (COM 和 LPT)」。Open "Device Manager" → expand "Ports (COM & LPT)".
  3. 記下 COM 編號（例：COM5）。Note the COM number (e.g. COM5).
開啟 Serial sessionOpen a Serial session
  1. MobaXterm → 點選 Session → 選 Serial。MobaXterm → click Session → choose Serial.
  2. 「Serial port」選剛剛查到的 COM port。For "Serial port", pick the COM port you just found.
  3. 「Speed (bps)」設為 115200（如圖）。Set "Speed (bps)" to 115200 (as shown).
  1. 按 OK 連線。Click OK to connect.
  2. 即可看到平台開機 log；最後出現 Qualcomm Linux ...exmp-q911 ttyMSM0 與 login 提示，代表已進入系統。You'll see the platform's boot log; the final Qualcomm Linux ...exmp-q911 ttyMSM0 and login prompt mean the system is up.
若 UART 無法使用，依以下內容處理If UART doesn't work, follow these
Windows

若 UART 線接上但裝置管理員沒有出現 COM port，或裝置出現在「其他裝置」並帶黃色驚嘆號，代表驅動尚未安裝。 If the UART cable is connected but no COM port appears in Device Manager, or the device shows under "Other devices" with a yellow exclamation mark, the driver isn't installed yet.

  1. 下載並解壓 driver 資料夾。Download and extract the driver folder.
  2. 裝置管理員 → 對該裝置按右鍵 → 「更新驅動程式」。Device Manager → right-click the device → "Update driver".
  3. 選「在我的電腦上瀏覽驅動程式」→ 指向 driver 資料夾，並勾選「包含子資料夾」→ 下一步。舉例如圖Choose "Browse my computer for drivers" → point to the driver folder, check "Include subfolders" → Next. Example shown below.
Ubuntu

若遇到 UART 無法使用請進行以下步驟。 If UART doesn't work, follow these steps.

  1. sudo dmesg | tail -20，確認是否為 brltty services 佔用問題。sudo dmesg | tail -20 to check whether brltty services are holding the port.
  1. 移除 sudo apt remove brltty。Remove it: sudo apt remove brltty.
  2. 再使用 minicom 連線看看 sudo minicom -D /dev/ttyUSB0 -b 115200。Try connecting with minicom again: sudo minicom -D /dev/ttyUSB0 -b 115200.
